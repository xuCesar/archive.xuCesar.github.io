关于版本控制

在我个人看来，版本控制就是一个实体拥有多个状态的可更改的副本。所谓实体，就是真实存在于本地环境下的文件。所谓副本，就是存在于远端服务器下的节点镜像文件。

本地文件通过版本控制工具提交到远端，那么远端就会将你的本地文件在当前时间节点下的内容以及状态保存下来。当你有新的改动提交之后，远端则会在上一节点的基础上进行更新。因此，你可以很清楚的查看文件的更改记录。

名词理解（仅个人）

工作区：当前操作的文件目录。

分支: 简单理解，把git比作一棵树的话，那么分支就是就是树干树枝了。每个树枝都有它自己的形态，如果某个树枝不好看，可以修剪打理，不会影响到其他的树枝。分支就是拥有自己的状态的副本。

暂存区(stage): 暂存区顾名思义，提交的文件被临时存放的地方。

命令总结

git init: 初始化仓库

git add <file>: 添加某个文件到暂存区

git add .: 将所有修改添加到暂存区

git commit -m '描述'：将暂存区的内容提交至本地仓库

git commit -a -m '描述'： 将工作区、暂存区的内容一起提交至本地仓库
