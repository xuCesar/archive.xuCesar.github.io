版本号的意义

发表于 2018-10-14 | 分类于 语义化版本 | 阅读次数

当我们在开发一个产品、或者网站的时候，可能需要使用层出不穷的类库。随着技术的发展，需求的增加，类库的更新也是一天一小更，三天一大更…由此，层出不穷的类库会产生层出不穷的版本。版本的管理和差异对项目的影响也越来越大。

概述

通常情况下，我们使用npm或者yarn来管理这些依赖包（package）。项目依赖（包括生产、开发甚至测试的依赖库）的版本信息以json的形式存储在package.json中。

Package.json

package.json文件被称为描述文件，它包含了所有需要共享的代码，以及描述包信息，易于开发人员对项目进行管理，以及更加友好的可移植性。

Package-lock.json

使用npm install命令安装依赖包时，自动生成的安装描述文件。官方解释如下：

package-lock.json is automatically generated for any operations where npm modifies either the node_modules tree, or package.json. It describes the exact tree that was generated, such that subsequent installs are able to generate identical trees, regardless of intermediate dependency updates.

大致的意思是说，package-lock.json文件是由于npm修改（增加，删除、更新）node_modules内的模块时自动生成的。它精确描述了你所使用的模块信息，以方便项目移植。

Lock.json

lock.json和package-lock.json的功能相似，只不过它是由yarn生成的，就不再细说。

版本号格式

版本号有着统一的格式，即：主版本号.次版本号.修订号，它们的传递规则如下：

  • 主版本号：不兼容的api修改
  • 次版本号：新增功能并向后兼容
  • 修订号： 修复bug并向后兼容
